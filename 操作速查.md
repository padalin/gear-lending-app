## 三、常做的事，怎麼做

### 新增 / 編輯器材
網站登入 → 後台 →「編輯器材內容」→ 列表裡新增或點器材編輯 → 存檔。（批次也有：批次新增、批次編輯）

### 加一個新的管理員（重要：兩個名單都要改）
1. 在 `src/adminEmails.js` 加上對方 email（小寫）。
2. 在 `firestore.rules` 的 `isAdmin()` 也加上同一個 email。
3. 重新部署：
   ```bash
   npm run build
   firebase deploy --only hosting
   firebase deploy --only firestore:rules
   git add -A && git commit -m "add admin xxx" && git push
   ```
> 只改前端不改規則 → 對方進得了後台但存不了東西；只改規則不改前端 → 反之。**一定兩邊一起改。**

### 本機測試（安全，連測試資料庫）
```bash
cd ~/"Library/Mobile Documents/com~apple~CloudDocs/_器材室管理/gear-lending-app running"
npm run dev
```
開 localhost，借/還/後台隨便測，動到的是 gear-lending-dev2，不碰正式資料。

### 部署網站到正式
```bash
npm run build
firebase deploy --only hosting
```
完成後開 https://gear-lending.web.app，頁尾版本號會更新成當下時間。

### 修改安全規則
改 `firestore.rules` → `firebase deploy --only firestore:rules` →（先在 dev2 測再上正式更保險：`firebase use gear-lending-dev2` 試完再 `firebase use gear-lending`）。

### 存檔備份（版控）
```bash
git add -A && git commit -m "說明這次改了什麼" && git push
```

## 四、雷區 / 注意

- **兩個管理員名單要同步**：`src/adminEmails.js` 和 `firestore.rules` 的 `isAdmin()`。
- `npm run dev:prod` 會連**正式**資料庫，平常別用，測試一律 `npm run dev`。
- 部署正式前，先 `npm run dev` 點過再 `build`。
- `gear-lending` 和（已刪的）`gear-lending-app` 名字很像；資料一直都在 **gear-lending**。
- 借/還是**公開免登入**的設計；目前借用人姓名/電話任何人都讀得到（已知限制，見下）。
- **專案放 iCloud（多裝置共用）**：換裝置工作前，先確認資料夾已「下載到本機」（不是僅雲端）；`node_modules` 不必跨機同步，每台各自 `npm install` 即可。若 `npm run dev`/`build` 出現找不到檔案的怪錯，跑 `rm -rf node_modules && npm install` 重裝。`cd` 指令用 `~/"Library/Mobile Documents/..."` 這種寫法，換哪台 Mac 都通用。
